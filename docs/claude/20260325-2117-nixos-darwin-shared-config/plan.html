Cross-Platform NixOS/Darwin Config

IMPLEMENTATION PLAN — dendritic pattern + flake-parts

Build a single public Nix flake that fully configures a Linux desktop, Linux laptop, and macOS laptop. Niri + Noctalia for the Linux desktop, shared dotfiles via home-manager and wrapper-modules, secrets via sops-nix. One command sets up a fresh machine; a QEMU VM proves it.

Key Trade-off: wrapper-modules vs niri-flake. wrapper-modules is less mature but produces portable derivations fitting the dendritic pattern. If issues arise, niri.nixosModules.niri from niri-flake is the battle-tested fallback.
Neovim: symlink + extraPackages. 40+ lazy.nvim plugins stay managed by lazy.nvim. Nix provides LSP servers and build tools on PATH via extraPackages, solving NixOS FHS issues without converting plugins to Nix.
Config in native files. All application config stays in .bash, .lua, .toml files. Nix modules do minimal wiring — a single source line, an xdg.configFile symlink. No inlining in Nix strings.
